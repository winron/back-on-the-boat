# back-on-the-boat — Codebase Guide

**回到船上** is a Progressive Web App for mastering HSK 1–6 Chinese vocabulary, grammar, sentences, dialogues, and reading passages. It runs entirely in the browser — no backend, no API, no auth.

---

## Tech Stack

| Layer | Technology |
|---|---|
| Framework | Next.js 16.2.1 (see AGENTS.md — has breaking changes) |
| UI | React 19 + Tailwind CSS v4 |
| Persistence | Dexie (IndexedDB) |
| SRS engine | ts-fsrs 5.x (FSRS algorithm) |
| Stroke order | hanzi-writer 3.x |
| PWA / offline | @serwist/next 9.x |
| Language | TypeScript (strict) |

> **Next.js 16** uses `--webpack` flag for dev/build (see `package.json` scripts). Read `node_modules/next/dist/docs/` before touching Next.js config or APIs.

## Key Conventions

### All pages are client components
Every page and component uses `"use client"`. There are no server components, server actions, or API routes.

### Path alias
`@/*` resolves to `./src/*`. Always use `@/` imports, not relative ones across directory boundaries.

### Data loading
Use `data-loader.ts` helpers — they fetch from `/data/hsk{N}-{type}.json` and cache in memory. Never read data files directly.

### Database
All persistence goes through `src/lib/db.ts` (Dexie). Do not add new stores without bumping the version and defining the migration:

```ts
db.version(3).stores({ /* existing + new stores */ });
```

Current stores (v2): `srsCards`, `settings`, `dailyStats`, `displaySettings`, `pageLevels`.

### SRS / cards
- Card IDs follow the pattern `hsk{N}-{itemId}` (e.g. `hsk1-word-042`).
- `module` field identifies which page owns the card: `"characters" | "grammar" | "sentences"`.
- A card is "mastered" when `bestGrade >= 3`.
- Use `getDueCards(module, limit, idPrefix)` and `getNewCards(module, limit, idPrefix)` — do not query `db.srsCards` directly in pages.

### Styling
- Tailwind v4 — theme tokens are declared in `globals.css` with `@theme`, not in a config file.
- Dark mode is forced via `dark` class on `<html>`. Do not add light-mode variants unless explicitly requested.
- Per-tab accent colors use `.tab-color-{1-5}` CSS classes (pink, yellow, blue, cyan, purple). Add new tabs by extending this pattern.
- Max content width: `max-w-lg` (32rem / 512px), centered with `mx-auto`.

### Display settings
Use the `useDisplaySettings` hook (context) to read `showPinyin` and `showEnglish`. Pass them into `<TrilingualLabel>` or `<PinyinDisplay>`. Do not read from Dexie directly in components.

### Audio (TTS)
Use `<AudioButton text="..." />` for pronunciation buttons. Do not call `speak()` directly in event handlers without first checking `isTtsSupported()`.

### PWA / service worker
- Service worker is **disabled in development** (`NODE_ENV === "development"`).
- `sw.ts` is compiled to `public/sw.js` at build time. Do not edit `public/sw.js` directly.
- Run `npm run build` to test service worker behavior.

## Development Workflows

### Install & run
```bash
npm install
npm run dev          # Next.js dev server (webpack mode)
```

### Regenerate content data
```bash
npm run prepare-data  # Runs scripts/prepare-data.ts via tsx
```
This overwrites all `public/data/hsk{N}-*.json` files.

### Build
```bash
npm run build        # Production build (webpack mode, Serwist enabled)
npm run start        # Serve production build locally
```

### Lint
```bash
npm run lint         # ESLint 9 (eslint-config-next + TypeScript rules)
```
No test runner is configured.

## Data Shape Quick Reference

### HskWord (vocab)
```ts
{ id, simplified, traditional?, pinyin, meaning, hskLevel, partOfSpeech?,
  frequency?, unitIndex, unitName, exampleSentence?, examplePinyin?, exampleMeaning? }
```

### GrammarPattern
```ts
{ id, hskLevel, title, structure, explanation,
  examples: [{ chinese, pinyin, english }] }
```

### SentenceExercise
```ts
{ id, hskLevel, targetSentence, targetPinyin, targetMeaning, wordBank: string[] }
```

### Dialogue
```ts
{ id, hskLevel, title, context?, lines: [{ speaker, chinese, pinyin, english }] }
```

### ReadingPassage
```ts
{ id, hskLevel, title, titleZh, type: 'short'|'story',
  paragraphs: [{ chinese, pinyin, english }], vocabHighlights: string[] }
```

### SrsCardState
```ts
{ id, module, due, stability, difficulty, elapsed_days, scheduled_days,
  learning_steps, reps, lapses, state (0–3), last_review?, bestGrade? }
```
States: `0=New, 1=Learning, 2=Review, 3=Relearning`

## Architecture Notes

- **No server-side rendering** — all data fetching happens client-side via `fetch()` in `data-loader.ts`.
- **Progressive unlock** — users work through HSK 1 → 6 sequentially per page; `useUnlockedLevel` derives the current max from mastered cards.
- **FSRS with fuzz** — `enable_fuzz: true` in `generatorParameters()` adds scheduling variability. Keep this setting.
- **Geist fonts** loaded via `next/font/google` in the root layout. Do not add other fonts without updating CSS variables.
- **suppressHydrationWarning** on `<html>` is intentional — avoids hydration mismatch from forced dark class.
